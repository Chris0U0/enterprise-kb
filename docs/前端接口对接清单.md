# 前端接口对接清单

本文档依据当前 `frontend/` 页面与共享组件中的数据需求整理，并与仓库内《项目架构与功能说明》中的后端能力对齐。**路径与命名可与实际 FastAPI 路由微调，但字段含义建议保持一致**，便于一次性替换 Mock / `localStorage` / `project-registry`。

---

## 0. 全局约定

| 项 | 说明 |
|----|------|
| **Base URL** | 例如 `https://api.example.com` 或开发环境 `http://localhost:8000` |
| **API 前缀** | 建议统一 `/api/v1`（与架构说明中示例一致） |
| **鉴权** | `Authorization: Bearer <access_token>`；未登录返回 `401` |
| **内容类型** | JSON 使用 `Content-Type: application/json`；上传使用 `multipart/form-data` |
| **分页（列表类）** | Query：`page`（从 1 开始）、`page_size`（默认 20，上限 100） |
| **排序** | 可选 `sort`（如 `created_at`）、`order`（`asc` \| `desc`） |
| **项目隔离** | 凡与项目相关的资源，建议显式传 `project_id`（路径或 Query），后端做权限校验 |
| **错误体（建议）** | `{ "detail": "人类可读说明", "code": "ERROR_CODE", "field_errors": { ... } }` |
| **时间** | ISO 8601 字符串，UTC 或带时区，如 `2026-04-19T10:00:00+08:00` |

**前端涉及页面/模块**：全局 `AuthProvider`、`fetch` 封装、React Query（若引入）的 `queryKey` 建议包含 `projectId`。

## 1. 认证与用户会话

**对应前端**：`src/lib/auth-context.tsx`（当前为 Mock + `localStorage`）、`src/app/login/page.tsx`。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/auth/login` | 登录，返回 Token 与用户信息 |
| `POST` | `/api/v1/auth/logout` | 登出（可选：服务端撤销 refresh token） |
| `GET` | `/api/v1/auth/me` | 当前用户信息与角色（进入应用时刷新会话） |
| `POST` | `/api/v1/auth/refresh` | 刷新 Access Token（若采用双 Token） |

### 1.1 `POST /auth/login`

**Request Body（JSON）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `email` | string | 是 | 邮箱或企业账号 |
| `password` | string | 是 | 密码 |

**Response 200（示例）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `access_token` | string | JWT 或opaque token |
| `token_type` | string | 一般为 `Bearer` |
| `expires_in` | number | 秒 |
| `user` | object | 见下表 `User` |

**`User`（与前端 `auth-context` 对齐）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string \| number | 用户 ID（前端路由与成员列表可统一用 string） |
| `email` | string | 邮箱 |
| `name` | string | 展示名 |
| `role` | string | 枚举：`Admin` \| `Editor` \| `Viewer`（与现有 `PageGate` / 按钮禁用逻辑一致） |

### 1.2 `GET /auth/me`

**Request**：无 Body（Header 带 Bearer）。

**Response 200**：`User` 对象（可附加 `avatar_url`、`permissions[]` 等扩展字段）。

## 2. 项目与成员

**对应前端**：`src/app/projects/page.tsx`（列表）、`src/app/projects/[id]/page.tsx`（详情）、`src/data/project-registry.ts`（待替换为接口）、`src/hooks/use-project.ts`、`src/app/projects/[id]/layout.tsx` + 子路由 `knowledge` / `qa` / `artifacts` / `collab`。

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/projects` | 项目列表（工作台「项目聚合看板」） |
| `POST` | `/api/v1/projects` | 新建项目（「新建项目」按钮） |
| `GET` | `/api/v1/projects/{project_id}` | 项目详情（含健康度、阶段、引导状态等） |
| `PATCH` | `/api/v1/projects/{project_id}` | 更新项目元数据 |
| `DELETE` | `/api/v1/projects/{project_id}` | 删除项目（可选，需 Admin） |
| `GET` | `/api/v1/projects/{project_id}/members` | 成员列表 |
| `POST` | `/api/v1/projects/{project_id}/members` | 邀请/添加成员 |
| `PATCH` | `/api/v1/projects/{project_id}/members/{user_id}` | 变更角色 |
| `DELETE` | `/api/v1/projects/{project_id}/members/{user_id}` | 移除成员 |

### 2.1 `GET /projects`（Query）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `q` | string | 否 | 名称/描述模糊搜索（对应项目列表页搜索框） |
| `page` | number | 否 | 分页 |
| `page_size` | number | 否 | 分页 |

**Response**：列表项建议包含前端卡片所需字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 项目 ID |
| `name` | string | 名称 |
| `description` | string | 简介 |
| `phase` | string | 阶段文案，如「开发联调」 |
| `member_count` | number | 成员数 |
| `document_count` | number | 文档数 |
| `health` | string | 如 `good` \| `warning` \| `critical` |
| `last_update_at` | string | 最近活动时间 |
| `pending_summary` | string | 一行摘要，如「2 待办 · 1 审批中」 |

### 2.2 `POST /projects`

**Request Body**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 项目名称 |
| `description` | string | 否 | 描述 |

**Response 201**：新建后的 `Project` 对象（含 `id`）。

### 2.3 `GET /projects/{project_id}`

**Response**：`Project` 扩展字段（供仪表盘与时间轴）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string |  |
| `name` | string |  |
| `phase` | string |  |
| `health` | object | 如 `{ "progress": 85, "risk": 12, "quality": 92 }`（与前端 HealthIndicator 一致时可映射） |
| `timeline` | array | 可选；里程碑 `{ date, event, status }[]` |
| `last_report_excerpt` | string | 可选；AI 周报摘要（`ProactiveSummaryCard` 初始正文） |
| `onboarding` | object | 见 **2.4** |

### 2.4 新成员引导 `onboarding`（`ProjectOnboarding`）

与 `src/data/project-registry.ts` 中 `ProjectOnboardingFlags` 对齐：

| 字段 | 类型 | 说明 |
|------|------|------|
| `has_uploaded_doc` | boolean | 是否已上传文档 |
| `has_indexed_knowledge` | boolean | 知识库是否已就绪 |
| `has_tried_qa` | boolean | 是否已试用问答/Copilot |
| `has_viewed_risk_or_report` | boolean | 是否已查看风险或报告 |

可由后端根据文档表、索引任务、问答日志聚合计算；前端只读展示。

## 4. 检索与问答（含流式）

**对应前端**：`src/app/copilot/page.tsx`、`src/hooks/use-search-stream-mock.ts`（待替换为真实 SSE）、`src/app/projects/[id]/qa/page.tsx`、`src/app/projects/[id]/qa/[sessionId]/page.tsx`、`src/components/copilot/*`。

架构说明已包含：

- `POST /api/v1/search/` — 统一检索
- `GET /api/v1/search/stream` — SSE

### 4.1 `POST /search`（非流式或一次性结果）

**Request Body**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | 是 | 用户问题 |
| `project_id` | string | 否 | 与 URL `?projectId=` 一致 |
| `session_id` | string | 否 | 多轮会话 ID |
| `strategy` | string | 否 | 如 `vector_rag` \| `agentic_rag` \| `multi_agent` \| `graph_enhanced`；不传则后端路由 |
| `document_ids` | string[] | 否 | 限定检索范围 |
| `complex` | boolean | 否 | 对应前端「复杂问题」勾选，供路由参考 |

**Response 200**

| 字段 | 类型 | 说明 |
|------|------|------|
| `answer` | string | 最终回答 |
| `citations` | array | 引用列表，见 **4.3** |
| `steps` | array | 可选；检索步骤摘要（替代 Mock 进度） |
| `session_id` | string | 回写会话 |

### 4.2 `GET /search/stream`（SSE）

**Query（或 POST Body，二选一由后端定）**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | 是 | 用户问题 |
| `project_id` | string | 否 |  |
| `session_id` | string | 否 |  |
| `strategy` | string | 否 |  |

**SSE 事件类型（与架构说明一致，供前端订阅）**

| 事件名 | 说明 |
|--------|------|
| `step` | 检索/路由步骤更新 |
| `thinking` | 中间推理（可选） |
| `chunk` | 答案增量 token |
| `citation` | 单条引用 |
| `done` | 结束 |
| `error` | 错误信息 |

前端需将 `useSearchStreamMock` 替换为 `EventSource` 或 `fetch` 流读取，并驱动 `SearchProgressPanel` + 正文区。

### 4.3 引用结构 `Citation`（`CitationList`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一 ID |
| `label` | string | 展示文案，如「需求文档V2.pdf · 第4页」 |
| `document_id` | string | 可选 |
| `href` | string | 可选；前端跳转知识库或带锚点，如 `/knowledge?projectId=1&doc=...` |
